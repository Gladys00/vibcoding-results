# Excel 导入预览问题对应处理逻辑

本文档记录 V1.3.2.2 中 Excel 导入预览页“问题”和“处理建议”的现有逻辑。

## 数据来源

预览页的问题列表来自 `assets/app.js` 中 `parseWorkbookRows()` 生成的 `parsedImport.warnings`。

每条问题结构包含：

- `level`：问题级别。
- `location`：Sheet 和行号。
- `message`：问题描述。
- `suggestion`：处理建议。

预览页只展示当前已选择 Sheet 对应的问题。

## 已确认的问题类型

### 未识别到台账明细字段

触发条件：

- 当前行未识别到品名、数量、单价或金额等可形成台账明细的数据。

当前处理：

- 该行在解析阶段被跳过。
- 不写入项目、台账、收付款或发票。

预览问题：

- `未识别到台账明细字段`

处理建议：

- `已跳过该行`

### 导入文件存在欠款金额

触发条件：

- Excel 行中识别到欠款金额。

当前处理：

- 欠款金额只作为校验参考。
- 不写入台账金额。
- 不直接参与收入、支出、毛利、应收、应付统计。
- 系统欠款仍按台账金额和收付款流水重新计算。

预览问题：

- `表格欠款金额为 ...，系统欠款金额会按台账金额和收付款流水重新计算`

处理建议：

- `该列仅作为校验参考，不写入台账金额；本行台账金额为 ...`

### 存在付款日期但付款金额为空

触发条件：

- 当前行存在付款日期。
- 对应付款金额为空或无法解析为有效数字。

当前处理：

- 系统仍生成一条收付款记录。
- 金额为 `0`。
- 状态为 `待核对`。

预览问题：

- `存在付款日期但付款金额为空`

处理建议：

- `导入后在收付款中补齐金额`

## 不属于当前逻辑的能力

当前版本未实现：

- 在预览页直接编辑 Excel 解析结果。
- 在预览页直接删除单条解析结果。
- 在预览页确认后自动修改源 Excel 文件。
- 服务端校验。

## 维护注意

如果后续新增问题类型，应同步维护：

- `parseWorkbookRows()` 中的 warning 生成逻辑。
- 导入预览页的问题展示。
- 导入修正页的处理选项。
- 本文档。
