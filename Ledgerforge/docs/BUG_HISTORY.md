# Bug 历史记录

## 已确认历史缺陷

### BUG-001

- 发现版本：V1.3.2.1
- 修复版本：V1.3.2.2
- 状态：已修复
- 问题描述：Excel 导入向导仍保留“项目归属确认”步骤，与用户拒绝在导入时进行项目归集的流程不一致。
- 影响范围：Excel 导入流程、步骤条、下一步跳转。
- 复现步骤：进入 Excel 导入向导，完成字段映射后继续下一步，会进入项目归属确认页。
- 根因：导入步骤数组和页面路由仍包含 `import-assignment`。
- 修复方式：从导入步骤和路由清单中移除归属步骤，并将旧 `#import-assignment` 路由兼容到预览页。
- 验证结果：已通过静态检查和 headless 浏览器自测，步骤条不再展示归属步骤，旧路由会进入导入预览保护页。
- 回归风险：低。统计归集模块未改动。
- 备注：

### BUG-002

- 发现版本：V1.3.2.1
- 修复版本：V1.3.2.2
- 状态：已修复
- 问题描述：Excel 导入预览页在没有真实解析数据时展示 mock 数据，容易让用户误以为是本次文件解析结果。
- 影响范围：Excel 导入预览页。
- 复现步骤：直接进入导入预览页，页面会展示示例项目、记录数和问题。
- 根因：预览页存在无 `parsedImport` 时的静态示例 fallback。
- 修复方式：移除 mock fallback；无解析数据时提示用户先上传 Excel；有解析数据时展示 Sheet、项目、台账、收付款、发票和问题。
- 验证结果：已通过 headless 浏览器自测，无解析数据时预览页不再展示 mock；真实 xlsx 上传后展示解析出的项目、台账、收付款、发票和问题。
- 回归风险：中。预览页展示内容增加，需关注表格横向滚动和数据为空状态。
- 备注：

### BUG-003

- 发现版本：V1.3.2.1
- 修复版本：V1.3.2.2
- 状态：已修复
- 问题描述：Excel 导入修正页和确认页存在 mock 数据或模拟导入 fallback，未完全基于真实解析数据。
- 影响范围：Excel 导入修正页、确认页、确认导入操作。
- 复现步骤：直接进入修正页或确认页，可看到示例问题或示例确认数据；无解析数据时仍可能触发模拟导入。
- 根因：修正页和确认页存在静态示例 fallback，确认导入 action 在无 `parsedImport` 时调用 `simulateImport()`。
- 修复方式：修正页改为读取 `parsedImport.warnings`；确认页改为读取过滤后的真实解析数据；无解析数据时禁止模拟导入并提示先上传解析文件。
- 验证结果：已通过 headless 浏览器自测，修正页展示真实 warning 和处理选项，确认页展示真实解析摘要，确认导入后进入结果页。
- 回归风险：中。需关注真实导入确认后导入结果页与项目台账数据一致性。
- 备注：
