# Excel 导入修正处理逻辑

本文档记录 V1.3.2.2 中 Excel 导入修正页“处理”字段的目的和现有逻辑。

## 页面目的

导入修正页用于让用户在写入系统前查看本次 Excel 解析产生的问题，并为每条问题选择一个处理方式。

当前版本中，“处理”字段的作用是：

- 明确本次导入对该问题的处理态度。
- 在导入确认页汇总展示。
- 帮助用户知道哪些问题会被跳过、哪些需要导入后补齐、哪些仅作为校验参考。

## 数据来源

修正页数据来自当前导入会话中的 `parsedImport.warnings`。

没有上传并解析 Excel 文件时，修正页不展示 mock 数据，只提示用户先上传文件。

## 处理选项生成逻辑

处理选项由 `assets/app.js` 中 `importWarningOptions()` 根据问题描述生成。

### 未识别到台账明细字段

可选处理：

- `已跳过该行`
- `返回源文件补齐后重新导入`

默认处理：

- `已跳过该行`

实际影响：

- 该行已经在解析阶段被排除。
- 当前选择不再改写解析结果。

### 欠款金额校验

可选处理：

- `仅作为校验参考`
- `返回源文件核对欠款后重新导入`

默认处理：

- `仅作为校验参考`

实际影响：

- 欠款金额不写入台账金额。
- 欠款金额不参与收入、支出、毛利统计。
- 系统欠款仍按台账金额和收付款流水计算。

### 付款日期存在但付款金额为空

可选处理：

- `导入后在收付款中补齐金额`
- `返回源文件补齐金额后重新导入`

默认处理：

- `导入后在收付款中补齐金额`

实际影响：

- 当前版本仍会生成一条金额为 `0`、状态为 `待核对` 的收付款记录。
- 用户需要导入后进入收付款记录补齐金额。

### 其它未分类问题

可选处理：

- `确认后继续导入`
- `返回源文件核对后重新导入`

默认处理：

- `确认后继续导入`

## 当前边界

当前“处理”字段不会：

- 直接修改台账明细。
- 直接修改收付款金额。
- 直接删除某条解析结果。
- 写回源 Excel 文件。
- 写入后端。

当前选择仅保存在前端当前导入流程状态中。刷新页面后该选择不会保留。

## 后续扩展方向

如后续要把“修正”升级为真正的数据修改能力，应补充：

- 对解析结果的行级编辑。
- 对单条问题的忽略、跳过、补齐、重新计算。
- 导入确认前的变更记录。
- 与操作日志的关联。
