# Ledgerforge 项目财务台账系统 MVP V1.3.2.2

## 打开方式

直接用浏览器打开：

```text
Ledgerforge/index.html
```

或启动本地静态服务器：

```bash
cd Ledgerforge
node dev-server.js
```

然后访问：

```text
http://127.0.0.1:4173/
```

本 MVP 使用：

- HTML
- Tailwind CSS CDN
- 原生 JavaScript
- LocalStorage 统一数据源

不包含：

- 后端
- 数据库
- API

## 已从 Prototype 升级的能力

- 所有运行时数据来自 `localStorage`，刷新页面后保持一致。
- 首次打开时从 `assets/seed-data.js` 初始化数据。
- `assets/data-store.js` 提供统一数据访问、CRUD、派生统计和操作日志。
- 支持项目、台账明细、收付款、发票、字段配置的新增、编辑、删除。
- 支持查询项目、客户、供应商。
- 危险操作统一二次确认。
- 表单提交有必填、长度、数字范围、日期合法性校验。
- 所有保存、更新、删除、导出、导入、重置操作有 Toast 反馈。
- 异步操作有按钮 Loading 和页面 Loading 状态。
- 操作日志记录时间、用户、模块、操作类型、对象、结果和失败原因。

## V1.1 修复范围

- 统一页面字段命名，例如收入、支出、毛利、往来单位、收支方向、发票状态。
- 所有统计卡片标题旁增加 `?` 帮助图标，Hover 显示统计口径。
- 弹窗点击遮罩不再关闭，避免误丢输入。
- 弹窗标题栏支持拖动，并限制在浏览器可视区域内。
- 弹窗关闭、取消、保存按钮均有明确行为。
- 危险操作继续使用统一二次确认。
- 台账明细金额统一按 `数量 × 单价` 自动计算，收入、支出、毛利、应收应付等统计均使用该计算口径。

## V1.2 功能完善

- 统计归集列表支持按归集项目、原始项目和客户搜索。
- 统计归集详情页新增归集台账明细，可搜索项目、往来单位、品名和规格。
- 归集台账明细支持直接编辑和删除，复用台账明细弹窗、二次确认、Toast 和操作日志。
- 统计归集项目列表支持删除归集项目，并将关联原始项目恢复为未归集。
- Excel 导入历史批次支持删除，工作台最近导入和导入历史页均已接入。

## V1.3 功能完善

- Excel 导入支持真实表格文件解析，支持 `.xlsx`、`.xls`、`.csv`、`.tsv`。
- 解析后可预览项目、台账明细、收付款、发票和字段映射结果。
- 确认导入后写入项目、台账、收付款、发票和导入批次，导入项目可在项目台账、统计归集、报表查询中搜索。
- 操作弹窗中的对象下拉和导出按钮改为中文显示，避免出现英文选项。

## V1.3.1 功能完善

- 字段配置页新增导入字段映射维护，支持搜索、新增、编辑、删除。
- 同一个表格列名只能维护一条映射，避免重复列名映射到不同系统字段。
- 导入解析优先使用维护映射，再使用系统内置别名自动识别。
- 欠款金额作为“仅校验”字段处理，不写入台账金额，不参与收入、支出、毛利计算。
- 自动识别规则收紧，避免“单位”误命中“单位名称”等短表头误匹配问题。

## V1.3.2 功能完善

- 字段配置可理解项收敛为显示、排序、是否必填。
- 将台账金额、收付款金额、发票金额、欠款金额统一命名，避免不同页面出现“金额”泛称。
- 项目台账、项目详情、统计归集、报表和导入页的金额类表头统一增加 `?` 口径说明。
- 已有本地数据会自动迁移旧字段名，例如“客户/主往来单位”“收款/付款”“开票/收票”“金额”。

## V1.3.2.1 本地修复

- 修复表格表头中 `台账金额` 帮助提示被裁切遮挡的问题。
- Excel 导入 Sheet 选择支持全选、取消全选、多选和取消选中，并在离开页面后保留选择状态。
- 字段映射页的 `调整` 改为调整导入字段映射，选择系统已有字段，不再打开新增字段弹窗。

## V1.3.2.2 功能完善

- Excel 导入向导去掉“项目归属确认”步骤，导入时不再要求用户做项目归集。
- 导入预览页仅展示本次真实解析出的 Sheet、项目、台账明细、收付款、发票和问题，不再展示 mock 示例。
- 导入修正页按真实解析问题展示处理方式，处理选择会在确认页汇总。
- 导入确认页按真实解析数据展示确认摘要，没有解析文件时不再触发模拟导入。

## 覆盖范围

原型覆盖文档中的 43 个页面、页签、弹窗和视角，并收敛为 6 个主工作区：

1. 工作台
2. Excel 导入
3. 项目台账
4. 统计归集
5. 报表查询
6. 字段配置

## UX 调整

没有严格按文档把 43 个项目做成 43 个平铺页面，而是采用更适合 SaaS 产品的结构：

- 导入流程改为连续向导。
- 项目详情使用页签承载台账明细、收付款、发票、导入来源。
- 报表的项目、客户、供应商、统计归集作为同一查询页的视角切换。
- 新增、编辑、导出、归集、字段配置使用弹窗完成。
- 工作台展示待处理事项、最近导入、未归集项目和异常数据。

## 主要可点击路径

```text
工作台
  -> Excel 导入
  -> 上传文件
  -> Sheet 选择
  -> 字段映射
  -> 导入预览
  -> 错误修正
  -> 导入确认
  -> 导入结果
  -> 项目详情
  -> 统计归集
  -> 报表查询
  -> 导出确认
```
